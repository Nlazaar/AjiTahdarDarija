==========================
3. MOBILE
==========================

3.1 Stocker le token dans un stockage sécurisé :
    - utiliser SecureStore (expo-secure-store)
    - remplacer AsyncStorage pour le token JWT
    - fallback vers AsyncStorage uniquement si SecureStore n’est pas disponible

3.2 Ajouter un anti-tampering :
    - vérifier l’intégrité du token
    - vérifier la signature JWT
    - invalider le token si modifié

3.3 Ajouter un système de logout automatique :
    - si le token expire
    - si le backend retourne 401
    - si la signature JWT est invalide

3.4 Ajouter un système de protection des écrans sensibles :
    - empêcher les captures d’écran (expo-screen-capture)
    - activer blur quand l’app passe en background

3.5 Ajouter un check de version :
    - si la version de l’app est obsolète → forcer la mise à jour
