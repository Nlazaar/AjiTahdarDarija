PROMPT PHASE 34 — Intégration TTS/ASR (Backend + Web + Mobile)
Je veux ajouter un système complet d’audio TTS/ASR pour la prononciation dans mon application.

==========================
1. BACKEND — AUDIO ENGINE
==========================

1.1 Créer un module AudioModule :
    - AudioService
    - AudioController

1.2 Ajouter un endpoint :
    GET /audio/tts?text=...
    → retourne un fichier audio (mp3 ou wav)
    → utiliser un provider TTS :
        - Google Cloud TTS
        - Azure Cognitive Services
        - ou Coqui TTS (open-source)

1.3 Ajouter un endpoint :
    POST /audio/asr
    → reçoit un fichier audio (base64 ou multipart)
    → utilise un provider ASR :
        - Whisper (OpenAI)
        - Vosk (open-source)
        - Azure Speech-to-Text

1.4 Ajouter un système de scoring :
    - comparer la transcription ASR avec la phrase attendue
    - calculer un score 0–100 basé sur :
        - distance de Levenshtein
        - erreurs phonétiques
        - omissions
        - rythme

1.5 Ajouter un endpoint :
    POST /audio/pronunciation-score
    Body : { expectedText, userAudio }
    → retourne :
        { score, errors, suggestions }
